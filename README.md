# Solve Together

친구와 꾸준히 코딩 테스트 문제를 풀고, 같은 문제에 대한 서로의 풀이를 비교하기 위한 저장소입니다.

## 진행 방식

1. 일요일에 다음 주 평일 문제 5개를 정합니다.
2. 문제마다 `날짜_플랫폼_문제이름` 형식의 폴더를 만듭니다.
3. 각자 GitHub 아이디를 파일명으로 사용해 풀이 코드를 올립니다.
4. 문제 폴더의 `README.md`에 결과와 짧은 회고를 남깁니다.
5. 주말에는 밀린 문제를 풀거나 가장 어려웠던 문제를 다시 풉니다.

기본 제한 시간은 40분입니다. 시간 안에 풀지 못해도 시도한 코드나 막힌 지점을 남깁니다.

## 폴더 구조

```text
problems/
└── 2026-07/
    └── week-03/
        ├── README.md
        └── 0715_PG_호텔-대실/
            ├── README.md
            ├── imjunhyeon.java
            └── friend.java
templates/
└── problem.md
```

플랫폼 이름은 짧게 통일합니다.

| 플랫폼 | 표기 |
| --- | --- |
| 백준 | `BOJ` |
| 프로그래머스 | `PG` |
| LeetCode | `LC` |
| SWEA | `SWEA` |

## 최소 규칙

- 기본 브랜치는 `main`이며, 처음에는 각자 직접 커밋합니다.
- 다른 사람의 풀이 파일은 수정하지 않습니다.
- 풀이 또는 힌트를 참고했다면 솔직하게 표시합니다.
- 정답 코드만 복사해서 제출하기보다, 막힌 이유를 한 줄이라도 기록합니다.
- 문제 본문 전체는 옮기지 않고 링크만 남깁니다.

## 파일 이름

풀이 파일은 `{github-id}.{확장자}` 형식을 사용합니다.

```text
imjunhyeon.java
friend.cpp
```

재풀이를 별도로 남기고 싶다면 파일명 뒤에 `_retry`를 붙입니다.

```text
imjunhyeon_retry.java
```

## 커밋 메시지

```text
solve: imjunhyeon 0715 호텔 대실
solve: friend 0715 호텔 대실
docs: 7월 3주차 문제 목록 추가
```

## 새 문제 추가하기

1. 해당 주차 폴더 아래에 문제 폴더를 만듭니다.
2. [`templates/problem.md`](templates/problem.md)를 복사해 문제 폴더의 `README.md`로 사용합니다.
3. 주차 `README.md`의 문제 목록을 갱신합니다.

처음 시작하기 전에 예시 파일명 `friend`를 실제 GitHub 아이디로 한 번만 바꾸면 됩니다.
