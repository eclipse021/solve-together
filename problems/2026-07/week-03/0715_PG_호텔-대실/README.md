## 짧은 회고

### 준현

- 접근: O(1500 * 1000) < 1억 이니, 시간(분) 단위로 반복문 계속 돌림
- 막힌 지점: 10분 뒤 입장 가능 조건 누락
- 다음에 기억할 것: 시간 복잡도 보고 접근하고 조건 잘 보기!

### 준희

- 접근: 예약을 시작 시간순으로 정렬하고, 최소 힙에 각 방의 "퇴실+청소 10분" 시각을 저장. 새 예약 시작 시각에 비는 방이 있으면 재사용, 없으면 방 추가. 힙 크기 = 필요한 최소 방 개수.
- 막힌 지점: 우선순위 큐(heapq)를 오랜만에 써서 사용법 감 잡는 데 시간이 좀 걸림(문법 찾아봄). 스케줄링 방식은 고민했지만 한 번에 통과.
- 다음에 기억할 것: heapq 사용법(heappush/heappop, 최소 힙) 익혀두기. 방 재사용을 최소 힙으로 관리하는 패턴 기억하기.
