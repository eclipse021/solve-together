## 짧은 회고

### 준현

- 접근: bfs 사용하되 우선순위 순서대로 이동하고, 제일 먼저 도착한 접근에 대해 해당 문자열을 성공 처리함 (접근 못할 경우는 실패 처리)
- 막힌 지점: 
  - 시간 초과 발생, 탐색 가능한 모든 경우를 queue에 넣은 것에서 set을 통해 해당 단계에 동일한 좌표는 안 가도록 변경
  - segment fault 발생, 마지막 경우(2500)에 다음 인덱스를 탐색하므로 최대 2500보다 1개 더 메모리를 잡았어야 함
- 다음에 기억할 것: 시간 복잡도 전체적으로 "정확히" 계산해 보고 들어가자!

### 준희

- 접근: 처음엔 BFS로 사전순 최단 경로를 찾으려 했으나 계속 시간초과. 힌트 2번(①BFS에 가지치기 필요 ②BFS 대신 그리디)을 얻고, 매 칸에서 사전순(d<l<r<u)으로 이동 가능하고 남은 횟수 안에 목적지 도달이 가능한(남은거리 ≤ 남은이동 && 홀짝 일치) 방향 딱 하나만 고르는 그리디로 해결.
- 막힌 지점: (1) BFS인데 같은 단계 방문 좌표 가지치기를 안 해서 상태가 폭발 → 시간초과. (2) 가지치기를 넣어도 여전히 초과 → 접근 자체를 BFS에서 그리디로 바꿔야 함을 힌트로 알게 됨.
- 다음에 기억할 것: "사전순 최소 경로 + 이동 횟수 고정" 문제는 매 칸 그리디 선택(거리·홀짝 가지치기)이 정석. BFS 쓸 땐 방문 체크 필수.
