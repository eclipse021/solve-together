## 짧은 회고

### 준현

- 접근: 다익스트라로 인지하고 접근했다기보단, BFS에 우선순위 큐를 섞은 버전으로 접근했음
- 막힌 지점:
  - 시작 부분과 가까운 지점에 대해 처리 부분이 막혔음
  - 시작 원소를 한 번에 전부 큐에 넣는 BFS 유형이 떠올랐고, 이 방식이면 시작 지점에서 최소를 보장할 수 있어 그대로 진행함
- 다음에 기억할 것: 맨 처음 시간 복잡도 계산, 문제 접근, 디버깅 등 발전한 듯! 다익스트라 구현 부분이 좀 비효율적이어서 다익스트라 구현 부분 복습하기!

### 준희

- 접근: 다익스트라 첫 문제라 스스로 접근을 못 잡음. 어떻게 풀지(다중 출발점 다익스트라 + 거리 대신 "경로상 최대 intensity 최소화")와 TODO가 뚫린 스켈레톤 코드를 받아 그 빈칸을 채우는 식으로 해결. 핵심은 `new_intensity = max(intensity[now], weight)` (일반 다익스트라의 `dist + weight` 자리를 max로 바꾼 것).
- 막힌 지점: 다익스트라 자체가 처음이라 골격(우선순위 큐, 방문 처리, 완화)을 떠올리지 못함. 특히 "최단 거리 합"이 아니라 "구간 중 최댓값을 최소화"하는 변형이라는 걸 스스로 못 잡음. 봉우리 도달 시 더 이상 진행하지 않는 처리도 힌트로 알게 됨.
- 다음에 기억할 것: 다익스트라 골격 직접 구현할 수 있게 복습(heapq, `if intensity[now] < cur: continue`, 완화 조건). 완화식만 문제에 맞게 바꾸면 최소합 외 다른 최적화(최대 최소화 등)도 같은 틀로 풀린다는 점.
