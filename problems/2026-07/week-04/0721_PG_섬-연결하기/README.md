## 짧은 회고

### 준현

- 접근: 최소 신장 트리 개념(어떤 문제를 푸는 것인지, 크루스칼/프림 알고리즘)을 공부하고 관련 문제를 처음 풀어봄. 접근은 비교적 구현이 익숙한 프림 알고리즘으로 접근했음
- 막힌 지점: pq의 pop 부분을 다익스트라 방법(arr가 더 작아야 갱신)으로 세움
- 다음에 기억할 것:
  - 다익스트라와 프림은 pop 판단만 다르다 (최소값인지 vs 값이 존재하는지)
  - MST는 시작점과 무관하므로 한 번만 돌리면 된다.

### 준희

- 접근: MST(최소 신장 트리) 문제. MST가 처음이었지만 힌트만 보고 크루스칼로 바로 해결. 간선을 비용 오름차순 정렬 후, 유니온-파인드로 사이클이 안 생기는 간선만 골라 합치며 비용 누적.
- 막힌 지점: 딱히 없었음. MST/유니온-파인드 개념만 힌트로 확인하고 구현은 무난하게 진행.
- 다음에 기억할 것: "모든 노드를 최소 비용으로 연결" → MST(크루스칼: 간선 정렬 + 유니온-파인드) 패턴. find에 경로 압축 적용해두기.
