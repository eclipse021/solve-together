## 짧은 회고

### 준현

- 접근: 화살을 한 발씩 처리하도록 DFS를 수행하고, `unordered_set`으로 이미 나온 화살 조합의 중복 탐색을 제거했다
- 막힌 지점: 어피치의 화살 정보가 고정이므로 최종 점수도 고정이라고 잘못 생각해, 점수 차이가 아닌 라이언의 점수만 최대화함...
- 다음에 기억할 것:
  - 조합 문제 풀이 방식 바꾸기
    - 현재 방식: 순열 DFS에서 `unordered_set`으로 중복된 조합 제거
    - 정석: `int start` 변수로 중복하지 않도록 DFS

### 준희

- 접근: DFS로 풀어야 한다는 판단까지는 스스로 했지만 구현이 안 나와서 해답을 봄. 과녁 0~10점을 순서대로 훑으면서 각 과녁마다 "어피치보다 1발 더 쏴서 뺏는다 / 포기한다" 두 갈래로만 분기하는 DFS. 종료 시점에 남은 화살은 전부 0점(인덱스 10)에 몰아주고 점수 차를 계산.
- 막힌 지점: DFS를 너무 오랜만에 써서 "위에 탈출 조건 if, 아래에 for문으로 재귀" 정도의 골격만 기억하고 있었고, 그것만으로는 이 문제를 못 풀었음. 특히 이 문제는 for 반복이 아니라 과녁마다 O/X 2분기로 넘어가는 형태라 기억하던 틀에 안 맞았던 게 컸음. BFS는 어느 정도 손에 익었는데 DFS는 아직 직관적으로 안 와닿음 → 연습 필요.
- 다음에 기억할 것:
  - DFS 골격은 "for문 순회"만 있는 게 아니다. 선택지가 2개뿐이면 `dfs(쏜다)` / `dfs(안쏜다)` 2분기로 내려가는 형태가 더 자연스럽다.
  - 백트래킹은 되돌리기까지가 한 세트. `result[start] = x` → `dfs(...)` → `result[start] = 0`. 남은 화살을 0점에 몰아준 것도 계산 후 되돌려야 함.
  - `final_result = result`는 참조 복사라 나중에 값이 같이 바뀜. 반드시 복사해서 저장.
  - 동점 처리: 점수 차가 같으면 낮은 점수 과녁(인덱스 10 → 0 방향)부터 비교해서 더 많이 맞힌 쪽 선택.
  - DFS 자체를 더 연습하기 (BFS 대비 아직 약함).
